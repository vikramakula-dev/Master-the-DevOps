2. Control Flow — Infra Examples

python — retry loop (Module 09 smoke-test rewritten)
import time, requests, sys

def wait_for_healthy(url: str, max_retries: int = 10, delay: int = 5) -> bool:
    """Poll URL until 200 OK or timeout — replaces the bash retry loop from M09."""
    for attempt in range(1, max_retries + 1):
        try:
            resp = requests.get(url, timeout=5)
            if resp.status_code == 200:
                print(f"✓ Healthy after {attempt} attempt(s)")
                return True
        except requests.ConnectionError:
            pass                        # service not yet up
        print(f"Attempt {attempt}/{max_retries}: not ready — retrying in {delay}s")
        time.sleep(delay)
    return False

if __name__ == "__main__":
    url = sys.argv[1]
    if not wait_for_healthy(url):
        print("ERROR: service never became healthy", file=sys.stderr)
        sys.exit(1)                     # non-zero exit = failure in CI (M02 discipline)

3. Functions, Modules & Error Handling

python — error handling done right
import boto3
from botocore.exceptions import ClientError, NoCredentialsError

def get_instance_ids(tag_name: str) -> list[str]:
    """Return EC2 instance IDs matching a Name tag."""
    ec2 = boto3.client("ec2")
    try:
        resp = ec2.describe_instances(
            Filters=[{"Name": "tag:Name", "Values": [tag_name]}]
        )
        return [
            i["InstanceId"]
            for r in resp["Reservations"]
            for i in r["Instances"]
        ]
    except NoCredentialsError:
        # catch specific — never bare `except:`
        print("ERROR: No AWS credentials found. Check ~/.aws/credentials or instance role.")
        sys.exit(1)
    except ClientError as e:
        # e.response['Error']['Code'] gives the AWS error code
        print(f"AWS error: {e.response['Error']['Message']}")
        raise               # re-raise for the caller to handle or propagate
⚠️ Never use bare except:

A bare except: catches everything including KeyboardInterrupt and SystemExit, masking bugs and making the script impossible to cancel. Always catch the most specific exception type. In automation code this matters more than in app code — a swallowed error that deletes the wrong S3 bucket is a very bad day.

4. Files: JSON, YAML, CSV

python — parse and patch a K8s manifest (CI image-tag bump)
import yaml, sys

def bump_image_tag(manifest_path: str, new_tag: str):
    """
    Read a K8s Deployment YAML, update the image tag, write it back.
    This is what CI tag-bumping really is — the yq shorthand does this under the hood.
    Module 17 (ArgoCD) teaser: this commit triggers a GitOps sync.
    """
    with open(manifest_path) as f:
        manifest = yaml.safe_load(f)    # safe_load: no arbitrary code execution

    containers = manifest["spec"]["template"]["spec"]["containers"]
    for c in containers:
        repo = c["image"].split(":")[0]    # strip old tag
        c["image"] = f"{repo}:{new_tag}"

    with open(manifest_path, "w") as f:
        yaml.dump(manifest, f, default_flow_style=False)
    print(f"Updated {manifest_path} → image tag: {new_tag}")

if __name__ == "__main__":
    bump_image_tag(sys.argv[1], sys.argv[2])

5. Key Libraries

python — subprocess (run kubectl/terraform from Python)
import subprocess

def run(cmd: list[str], check: bool = True) -> str:
    """
    Run a shell command. Pass a list — NEVER a string with shell=True
    if any part comes from user input (shell injection risk).
    """
    result = subprocess.run(
        cmd,
        capture_output=True,
        text=True,
        check=check    # raises CalledProcessError if returncode != 0
    )
    return result.stdout.strip()

# Good: list form — no shell injection possible
output = run(["kubectl", "get", "pods", "-n", "production"])

# Never do this with user-provided input:
# subprocess.run(f"kubectl get pods -n {namespace}", shell=True)  # ← injection risk

6. REST APIs with requests

python — GitHub API: list stale PRs + Slack notification
import requests, os
from datetime import datetime, timezone, timedelta

GH_TOKEN = os.environ["GITHUB_TOKEN"]       # never hardcode credentials (M03 rule)
SLACK_WEBHOOK = os.environ["SLACK_WEBHOOK_URL"]
REPO = "techitfactory/api"
STALE_DAYS = 7

def get_stale_prs() -> list[dict]:
    headers = {"Authorization": f"Bearer {GH_TOKEN}", "Accept": "application/vnd.github+json"}
    prs, page = [], 1
    cutoff = datetime.now(timezone.utc) - timedelta(days=STALE_DAYS)

    while True:                             # pagination — always handle it
        resp = requests.get(
            f"https://api.github.com/repos/{REPO}/pulls",
            headers=headers,
            params={"state": "open", "per_page": 100, "page": page},
            timeout=10
        )
        resp.raise_for_status()           # raises on 4xx/5xx
        batch = resp.json()
        if not batch:
            break
        prs.extend(pr for pr in batch
                   if datetime.fromisoformat(pr["updated_at"].replace("Z","+00:00")) < cutoff)
        page += 1
    return prs

def notify_slack(prs: list[dict]):
    if not prs:
        return
    text = f"*{len(prs)} stale PRs (no activity > {STALE_DAYS}d):*\n"
    text += "\n".join(f"• <{pr['html_url']}|{pr['title']}> by @{pr['user']['login']}" for pr in prs[:10])
    requests.post(SLACK_WEBHOOK, json={"text": text}, timeout=5)

notify_slack(get_stale_prs())

7. boto3 — AWS SDK for Python

boto3 uses the same credential chain as the AWS CLI (Module 10): environment variables → shared credentials file → instance role → container role. Never call boto3.client('ec2', aws_access_key_id=...) — let the chain work.

python — boto3 EC2 inventory + start/stop scheduler
import boto3, csv
from datetime import datetime

def ec2_inventory(region: str = "us-east-1") -> list[dict]:
    """Return EC2 inventory with name, state, type, and cost tags."""
    ec2 = boto3.client("ec2", region_name=region)
    paginator = ec2.get_paginator("describe_instances")  # paginators handle >1000 results
    rows = []
    for page in paginator.paginate():
        for r in page["Reservations"]:
            for i in r["Instances"]:
                tags = {t["Key"]: t["Value"] for t in i.get("Tags", [])}
                rows.append({
                    "id": i["InstanceId"],
                    "name": tags.get("Name", "—"),
                    "state": i["State"]["Name"],
                    "type": i["InstanceType"],
                    "project": tags.get("Project", "untagged"),  # cost audit
                })
    return rows

def save_csv(rows: list[dict], path: str):
    with open(path, "w", newline="") as f:
        w = csv.DictWriter(f, fieldnames=rows[0].keys())
        w.writeheader()
        w.writerows(rows)

if __name__ == "__main__":
    rows = ec2_inventory()
    out = f"inventory_{datetime.now().strftime('%Y%m%d_%H%M')}.csv"
    save_csv(rows, out)
    untagged = [r for r in rows if r["project"] == "untagged"]
    print(f"Saved {len(rows)} instances to {out}. Untagged: {len(untagged)}")

8. Automation Patterns — Health Check with Auto-Recovery

python — app_monitor.py (health check + restart + Slack alert)
import requests, subprocess, time, os, logging

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s",
    handlers=[
        logging.FileHandler("/var/log/app_monitor.log"),
        logging.StreamHandler()
    ]
)
log = logging.getLogger(__name__)

APP_URL     = os.environ["APP_URL"]
SLACK_URL   = os.environ["SLACK_WEBHOOK_URL"]
CONTAINER   = os.environ.get("CONTAINER_NAME", "api")
COOLDOWN    = 300           # seconds between auto-restarts (prevent flapping)
_last_restart = 0

def is_healthy() -> bool:
    try:
        r = requests.get(APP_URL, timeout=5)
        return r.status_code == 200
    except Exception:
        return False

def alert_slack(msg: str):
    try:
        requests.post(SLACK_URL, json={"text": msg}, timeout=5)
    except Exception as e:
        log.warning(f"Slack notify failed: {e}")   # don't crash if Slack is down

def restart_container():
    global _last_restart
    if time.time() - _last_restart < COOLDOWN:
        log.warning("Skipping restart — cooldown period active (prevents flapping)")
        return
    log.info(f"Restarting container {CONTAINER}")
    subprocess.run(["docker", "restart", CONTAINER], check=True)
    _last_restart = time.time()
    alert_slack(f"⚠️ Auto-restarted {CONTAINER} at {APP_URL}")

if __name__ == "__main__":
    log.info("Monitor started")
    while True:
        if not is_healthy():
            log.error(f"Health check FAILED: {APP_URL}")
            alert_slack(f"🚨 {CONTAINER} is DOWN at {APP_URL}")
            restart_container()
        time.sleep(30)

python — lock file pattern
import fcntl, sys

LOCK_FILE = "/tmp/aws_inventory.lock"

try:
    lock_fd = open(LOCK_FILE, "w")
    fcntl.flock(lock_fd, fcntl.LOCK_EX | fcntl.LOCK_NB)  # non-blocking exclusive lock
except BlockingIOError:
    print("Another instance is running. Exiting.")
    sys.exit(0)

try:
    # ... main script work ...
    pass
finally:
    fcntl.flock(lock_fd, fcntl.LOCK_UN)
    lock_fd.close()

import re, sys
from collections import Counter

LOG_PATTERN = re.compile(
    r'(?P<ip>\S+) .+ \[.+\] "(?P<method>\S+) (?P<url>\S+) \S+" (?P<status>\d{3}) (?P<size>\d+)'
)

            statuses[m.group("status")] += 1

Interview Questions

Click a card to reveal a strong candidate answer.

Q: When do you choose Python over Bash for automation?

Bash is ideal for: chaining CLI commands, simple file manipulation, short one-off scripts. I switch to Python when: the script is over ~50 lines (readability matters), I need to handle complex data structures (JSON, YAML parsing), I'm calling APIs or AWS SDK (boto3 is far better than CLI + jq for complex queries), I need real error handling, retry logic, or unit tests. Python is also the better choice when other engineers will maintain the script — it's more universally readable than advanced Bash.

Q: How do you safely use subprocess when user input is involved?

Always pass a list, never a string with shell=True. With shell=True, if a variable contains ; rm -rf / the shell will execute it. With a list, each argument is passed directly to the OS — no shell interpolation occurs. Also: validate inputs before building the command list (check allowed characters, use allow-lists not block-lists). If you need to redirect, use stdout=subprocess.PIPE rather than shell redirection characters in a string.

Q: How do you handle API pagination and rate limiting?

Pagination: always loop until the response returns an empty page or null next-page token. Never assume one call returns all results — GitHub, AWS, and most APIs cap at 100 items per call. Rate limiting: check the X-RateLimit-Remaining header (or equivalent), back off when it's near zero, implement exponential backoff with jitter on 429 responses. For boto3, use paginators (client.get_paginator(...)) — they handle AWS pagination automatically and are much cleaner than manual page-looping.

Q: How does the boto3 credential chain work?

boto3 looks for credentials in this order: (1) explicit kwargs to the client (aws_access_key_id= — avoid), (2) environment variables (AWS_ACCESS_KEY_ID), (3) shared credentials file (~/.aws/credentials), (4) AWS config file, (5) instance metadata service (EC2 instance role / ECS task role / Lambda execution role), (6) ECS container credentials. In practice: on a developer machine, use the config file or environment variables. On EC2/ECS/Lambda/EKS with IRSA, use the instance or task role — no credentials file needed or wanted.

Q: Walk me through writing an auto-remediation script and its risks.

The app_monitor.py pattern: (1) health check on a well-defined endpoint, (2) on failure, alert Slack immediately, (3) attempt restart only after a cooldown period to prevent flapping. The risks: flapping — if the root cause isn't fixed, the script restarts in a loop, potentially masking the real issue; cascade — auto-restarting a service that's down due to a database failure can make the database situation worse; alert fatigue — too many noise alerts and teams stop reading them. My rule: auto-remediate only for narrow, well-understood failure modes. Log everything, page a human for anything outside that envelope.
